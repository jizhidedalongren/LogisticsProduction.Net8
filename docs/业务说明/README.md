# 业务说明

本目录存放**业务相关说明文档**，如业务流程、业务规则、术语表、业务对象说明等。

## 文档规范

- 文件名使用**中文**，便于识别（如 `仓储业务流程.md`、`AGV调度规则.md`）
- 每个业务模块/场景单独一个文件
- 建议包含以下内容：
  - 业务背景与目标
  - 流程说明（可附流程图或时序描述）
  - 关键概念与术语
  - 涉及的接口 / 数据表
  - 业务规则与约束

## 与其他目录的关系

- [项目需求与设计](../项目文档/) — 技术架构与实现方案
- [待开发需求](../待开发需求/) — 尚未开发的需求；开发完成后，相关说明文档归档至此目录

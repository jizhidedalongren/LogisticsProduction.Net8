## 🛠️ 脚本工具

### 脚手架生成
- `scripts/scaffold-cn.ps1` — 中文交互式脚手架生成器（推荐）⭐
- `scripts/scaffold.ps1` — 英文交互式脚手架生成器
- `scripts/scaffold-feature.ps1` — 命令行脚手架生成器（核心脚本）

### 部署脚本
- `scripts/deploy.ps1` — 通用部署脚本（支持 IIS 和 Windows 服务）⭐
- `scripts/deploy-to-iis.ps1` — IIS 专用部署脚本（首次安装 + 更新 + 热更新）⭐

### 工具脚本
- `scripts/fix-utf8-bom.ps1` — 修复 PowerShell 脚本编码为 UTF-8 BOM
- `scripts/generate-entities.ps1` — SqlSugar 实体类生成工具

> 脚本位于项目根 `scripts/` 目录，需以管理员权限运行。
> 包含中文的 PowerShell 脚本均使用 UTF-8 BOM 编码，遇中文乱码运行 `.\scripts\fix-utf8-bom.ps1` 修复。

## 🎯 常见场景

### 我想开发新功能
1. 阅读 [开发指南.md](开发指南.md) 了解开发流程
2. 运行 `.\scripts\scaffold-cn.ps1` 生成代码骨架
3. 参考 [架构规范.md](架构规范.md) 遵循架构规范

### 我想接入多数据库
1. 阅读 [多数据库指南.md](多数据库指南.md) 了解实体存放与访问方案
2. 参考其中的「实施进度」了解已完成与待办

### 我想部署到生产环境
1. 阅读 [Windows部署指南.md](Windows部署指南.md) 选择部署方式
2. IIS 部署：`.\scripts\deploy-to-iis.ps1 -FirstInstall`
3. Windows 服务：`.\scripts\deploy.ps1 -Environment Production`

---

## 💡 提示

- 所有 PowerShell 脚本需以管理员权限运行
- 如遇"无法加载文件"错误：`Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process`
- 部署前请先在测试环境验证
- 遵循项目架构规范开发新功能
